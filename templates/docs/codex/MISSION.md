# Project Mission

Describe the durable product goal for this repository.

The checkpoint loop uses this file to judge progress and generate the next small batch of tasks. Keep it stable and specific.

## Mission

Build a polished, safe, useful project without broad rewrites or risky production changes.

## Success Looks Like

- The app or site is more useful and professional after each batch.
- Every task is small, reviewable, and build-verified.
- Risky work stays out of unattended loops.
- A person reviews branch progress before anything is merged to `main`.

## Hard Limits

- Do not merge to `main`.
- Do not deploy.
- Do not add secrets, credentials, API keys, or environment variables.
- Do not touch billing, DNS, auth, production backend systems, or production data shapes unless explicitly approved.
