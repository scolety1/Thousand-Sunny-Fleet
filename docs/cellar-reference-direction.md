# Cellar Reference Direction

The Cellar / Restaurant Demo Fleet has two different design reference sets because the demos have two different audiences.

## Guest-Facing Restaurant And Hospitality Websites

Use this set for public restaurant pages, wine lists, event inquiry pages, private dining pages, beverage pages, and full restaurant websites.

Reference URLs:
- https://www.dishoom.com/
- https://www.mujoatl.com/
- https://noburestaurants.com/
- https://boccalupoatl.com/
- https://www.goodasfolk.com/
- https://www.butcherandbee.com/

Use these references for:
- rich hospitality atmosphere
- editorial spacing
- restrained but memorable typography
- progressive disclosure
- beautiful first impression
- menus that feel browseable, not dumped onto one page
- mobile-first restaurant browsing
- strong sense of place and brand

Do not copy exact layouts, text, branding, menu items, images, or trade dress.

Core rule: public-facing pages should feel like real restaurant websites first and software demos second. The user should not be hit with every feature immediately. Show a beautiful first layer, then make details easy to find.

Use these references for:
- calm mobile hierarchy
- fast scanning

- compact task/action flows
- obvious tap targets
- day/shift structure
- internal tools that feel useful instead of overwhelming

Do not copy exact product UI, brand styling, icons, paid-product trade dress, or enterprise dashboard clutter.

Core rule: manager-facing pages should feel like tools someone can use during a shift. They should be simpler than a dashboard, quieter than a marketing site, and organized around one job at a time.

## Global Rule

The biggest current problem is information overload.

Do not show every feature, note, list, and control on the first screen.

Use:
- one primary job per screen
- short first impression
- buttons or tabs for secondary areas
- progressive disclosure
- clear navigation
- mobile-first layouts
- fewer giant all-in-one pages

Examples:
- A wine list QR page should show the wine list first.
- "Help me choose" should be an easy button, not the whole page.
- A manager brief should start with the shift summary, then separate 86s, tasks, staffing, notes, and prep into simple sections.
- A restaurant website should feel like a restaurant website, not a software dashboard.

Goal: beautiful, simple, usable demos that show what the finished customer version could look like.
