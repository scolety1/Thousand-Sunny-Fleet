# Deferred and Acceptance Boundary

Not implemented in Milestone 3:

- background queue watching, a daemon, service, scheduled task, or startup registration;
- automatic mission continuation, approval, TIM_REQUIRED response, completion, retry, merge, deployment, or production action;
- old Codex thread/turn resumption;
- multiple simultaneous governed missions;
- arbitrary command, executable, environment, repository, or product-repository bridges;
- plugins, credentials, package installation, broad network, or worker-tool network;
- generic reset/delete/force-complete/clear-queue/kill controls;
- another queue, result, admission, replay, approval, or recovery database.

Milestone 4 acceptance must not begin from this branch. The required next gate is independent Milestone 3 review, acceptance, and merge.
