# Deferred to Milestone 3

Milestone 2B deliberately does not add:

- background queue watching, persistent processes, or remote listeners;
- suspended/resumable interactive workers;
- arbitrary command, script, executable, environment, or repository bridges;
- broader approval policies, wildcard scopes, automatic reuse, or expiry extension;
- editable UI authority, mission, queue, verifier, admission, or receipt controls;
- product-repository execution or unsafe real-write proofs;
- plugins, credentials, deployments, package installation, merge, or auto-merge;
- parallel approval, revision, queue, execution, verification, or admission authorities.

Any Milestone 3 proposal must begin from the canonical ledger/mission/queue/lifecycle/verifier/admission owners and preserve old-run immutability and exact authority matching.
