# TSF V1 Demonstration Script

Target duration: 10–15 minutes after the full acceptance run has been completed and sealed.

1. Run human and JSON Doctor. Point out the stable labels, exact commit/worktree, path budget, owner/listener state, reconciliation counts, and exact next action.
2. Start production HQ Dispatch in the foreground. Show fixed loopback, one exact owner, disclosed roots, and second-instance rejection.
3. In the browser, preview a bounded TSF-local read-only request. Emphasize `PREVIEW_ONLY_NOT_AUTHORITY` and that no mission or worker exists yet.
4. Submit the governed mission. Show submission, mission, revision, run, result, and canonical queue identities.
5. Show the real app-server adapter receipt from the sealed acceptance run: child PID/start, thread, turn, exact response hash, clean child exit, and no orphan.
6. Show verifier identity/result, preservation path/hash, admission verdict, receipt ID/path/hash, caveats, and exact next action.
7. Run the deterministic Demo for approval, denial, and clarification choreography. Keep the fixture banner visible; do not call its worker a real app-server result.
8. Show denial produces no worker/revision. Show clarification links the response and creates a distinct revision/run. Show authority-changing clarification stops at a fresh approval request.
9. Replay exact submission and response records; show existing identities. Change content; show the conflict fails closed.
10. Show the sealed controlled-interruption receipt: real child observed before any terminal result, public Stop, `INTERRUPTED_PROCESS_GONE`, no automatic rerun, and source bytes unchanged.
11. Show explicit recovery created distinct mission/run/thread/turn/verifier/admission and an idempotent recovery receipt; old thread/turn were not resumed.
12. Stop the live server cooperatively. Re-run Doctor and prove no owner, listener, or owned child. End on the authority-denial and known-limitations panels.

If Codex control-plane service/authentication is unavailable during the acceptance run, label the real segment YELLOW and do not publish. A deterministic fixture can demonstrate choreography but cannot replace the required real proof.
