# Foreground Execution Contract

The relay invokes only the existing PowerShell queue executor with fixed paths and arguments. Processes use `shell:false`, `detached:false`, hidden windows, piped bounded output, a bounded timeout, and one tracked child handle. A second mission is rejected while a different mission is active. Server shutdown invalidates sessions, marks the local projection `INTERRUPTED`, and terminates and waits boundedly for the owned direct child.

The canonical lifecycle and app-server adapter remain responsible for native app-server identity and child/no-orphan evidence. HQ Dispatch does not fabricate that evidence. In the real proof, adapter instance `76bd2252-5dea-4e2e-8bf2-1e6feb8d91b9` owned child PID `7396`. The protocol completed, cooperative shutdown completed, the child exited, the PID was absent afterward, and no owned listener remained on loopback port `41737`.

No watcher, daemon, detached process, arbitrary shell bridge, product repository, plugin, credential, worker-tool network, push, merge, deployment, package installation, remote listener, or production mutation was introduced.
