# ORIGINAL_OPERATOR_INTENT_V1

Schema: `fleet/control/original-operator-intent.schema.v1.json`. Canonical implementation: `tools/TsfSemanticIntegrity.ps1`.

The closed contract contains the raw-request SHA-256, normalized evidence SHA-256, requested goal and deliverable, explicit and authority-bearing operations, repository/worktree, access/network request, prohibited operations, ambiguity status, source preview identity, version, and a canonical JSON identity hash.

Operation extraction is negation-aware and is combined with the existing Project Main Bot classification, allowed-read/write proposal, forbidden-action registry, worker-role registry, branch/worktree policy, and submission revalidation. It is a strengthening of the existing route, not a new execution authority.

Every preview is hashed and session-bound. Submission re-reads the stored preview, recomputes the request and intent contract, compares a fresh route projection, and then mission preparation independently recomputes the stored contract before a mission id or queue record can exist.
