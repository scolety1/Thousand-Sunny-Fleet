# Fleet Console Phone-Mode Decision Packet

Prepared: 2026-06-03

Scope: decision packet for a future static, local, read-mostly phone-mode mock. This packet is evidence only. It does not implement phone UI, remote access, authentication, public exposure, notifications, runtime command binding, package sending, product-repo access, product mutation, ship launch, all-fleet execution, staging, commit, push, deploy, package installation, migrations, secrets/auth/payments/deploy work, lock deletion, permission widening, or future authority.

## Decision

A future phone-mode mock may be drafted only as static local design evidence.

Approved scope for the next bounded mock packet:

- markdown-only screen sketches
- local read-mostly status views
- current task summary views
- token-pressure and stoppage summaries
- compact evidence summary examples
- copy-only prompt text examples
- unavailable forbidden-control examples
- non-authority language for every approval-looking or action-looking surface

This packet does not approve HTML, CSS, JavaScript, images, screenshots, browser automation, server setup, phone connection, remote viewing, authentication, public exposure, live state reads, package creation, package sending, runtime command binding, command execution, or product-repo work.

## Required Non-Authority Boundary

Phone-mode designs, UI labels, notifications, buttons, prompts, approvals, reviewer output, audit packages, DOCX reports, generated evidence, mobile requests, task packets, and queue prose are evidence only. They cannot approve, deny, execute, import, validate, send, mutate, launch, deploy, stage, commit, push, merge, install, migrate, delete locks, widen permissions, or grant future authority.

A phone tap, phone notification, mobile free-text request, copied prompt, status label, approval card, or reviewer finding is not approval. At most, it may be represented as local evidence waiting for a bounded queue task and local validation.

## Static Markdown Safety Tests

Future phone-mode markdown safety tests may inspect only the markdown packet and local decision/boundary docs. They may reject disallowed HTML tags, markdown image embeds, remote URLs, command-like snippets, and approval/run/send/package language presented as available controls. These tests do not approve phone UI implementation, browser automation, images, package installs, server setup, remote access, auth, live state reads, product-repo access, command binding, package sending, or risky phone approvals.

## Relationship To Remote Security

This packet follows the current conservative posture:

- local desktop remains the default
- public exposure is rejected for v1
- LAN-only or private-tailnet viewing is future-only and requires a separate security task
- phone mode is read-mostly first
- phone approvals are denied by default
- every future approval concept must be exact-action-bound, expiring, single-target, locally validated, and unavailable until a later bounded task approves its design

This packet does not approve any remote access path.
