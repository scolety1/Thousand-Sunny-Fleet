# Engine Blueprint - Nytheria

Generated by TSF Game Forge V1 from TSF-local intake only. Evidence only; not executable authority or approval.

## Intake Summary

- Game name: Nytheria
- Old names / aliases: Nytheris
- Genre: living fantasy world RPG / AI-assisted world simulator
- Camera/view style: text-first simulation cockpit, later map and scene views
- Target platform: local desktop/web prototype first
- Core promise: enter a high-magic, low-tech world that moves without you; you are not automatically the chosen one.

## Recommended Engine Style

- Data-driven simulation-first RPG cockpit.
- Start text-only with deterministic fixtures before UI, combat, graphics, or AI game-master layers.
- Treat lore as versioned evidence, approved canon, or adaptation proposal.

## Core Runtime Loop

1. Load world snapshot and canon registry.
2. Advance World Clock by one tick or turn.
3. Resolve faction goals, god/faith pressures, region changes, rumors, and timeline events.
4. Apply player action ledger effects.
5. Emit visible consequences, quest hooks, rumors, and changed world state.
6. Save deterministic snapshot for replay and tests.

## Data Model

- GameState: clock, regions, factions, gods, rumors, NPC memory, quests, player ledger, doom-clock state.
- CanonRegistry: root lore evidence, approved canon, adaptation proposals, source refs, alias mapping for Nytheris/Nytheria.
- EventQueue: scheduled timeline events, war moves, divine interventions, rumor mutations.
- PlayerActionLedger: choices, origin/race/class flags, faith acts, faction impacts, relationship impacts.

## Content Pipeline

- Store early content as plain markdown/json fixtures before editor tooling.
- Require canon tier for lore entries: evidence, approved canon, adaptation proposal, rejected.
- Keep old Nytheris lore as backbone evidence until Tim approves migration into Nytheria canon.

## Save/Load Model

- Snapshot GameState at turn boundaries.
- Keep player action ledger append-only for auditability.
- Add deterministic fixture saves for tests before any autosave UX.

## Event System

- Events have trigger, preconditions, effects, visibility, rumor output, and save/load serialization.
- Default Timeline / Doom Clock events can fire without player involvement.

## Quest / Story System

- Quests are views over world state, faction goals, rumors, NPC memory, and player ledger.
- Avoid authored story branches until canon tiers and world-state schema exist.

## World Clock

- Tick-based simulation clock with region, faction, faith, rumor, and timeline updates.
- Supports paused inspection, deterministic replay, and fixture snapshots.

## Simulation Boundaries

- Prototype one region, two factions, one god/faith pressure, three rumor seeds, and one doom-clock arc.
- No infinite procedural world, no full economy, no full combat, no multiplayer, no production AI GM.

## AI Boundaries

- AI can propose rumors, summaries, NPC recollections, or adaptation proposals.
- AI must not rewrite approved canon, root lore, player ledger, or save-state truth without Tim approval.
- AI output remains suggestion/evidence until validated into structured state.

## Required Modules

- World Clock
- Faction Simulator
- God/Faith System
- Rumor System
- NPC Memory
- Origin/Race/Class Consequence System
- Default Timeline / Doom Clock
- Region State
- Encounter Director
- Lore Canon Layer
- Player Action Ledger

## Testing Strategy

- Golden fixture for one world-clock tick.
- Faction turn deterministic tests.
- Rumor mutation snapshot tests.
- NPC memory append/read tests.
- Save/load round-trip tests.
- Canon tier validation tests.

## Prototype Milestone Plan

1. Text-only world clock prototype.
2. Faction turn simulator.
3. Rumor propagation simulation.
4. NPC memory toy model.
5. Default Timeline / Doom Clock.
6. Origin-based opening state generator.
7. Lore canon registry.

## What Not To Build Yet

- Full game engine.
- Real-time AI dungeon master.
- Full combat engine.
- Multiplayer.
- Infinite procedural lore.
- Graphics pipeline.
- Production content editor.
- Any system that rewrites canon without Tim approval.
